# Portfolio MVP Build Brief for Codex

## Project Owner

Yat Tin Lee  
Sydney-based multidisciplinary designer  
5 years industrial design experience  
4 years project management experience  
Currently expanding into UI/UX and research through master's study

## Goal

Build a modern, sustainable personal portfolio website that can be sent to employers.

The website should also support PDF portfolio export pages using the same project content.

## Current Content Status

Only Touch of Time is prepared with real selected images and a detailed project manifest.

Other projects currently use placeholders and will be updated later.

Codex must build the website so placeholder projects can be replaced later without restructuring the site.

## Design Direction

Editorial designer portfolio with a small touch of playful Gen-Z personality.

The style should be:

- modern
- clean
- image-led
- editorial
- professional
- slightly playful
- employer-friendly
- easy to scan

Avoid heavy animation and avoid overcomplicated visual effects.

## Important Content System Rule

The portfolio is project-based and tag-filtered.

Do not structure the website around fixed category folders.

Projects can have multiple tags.

Example tags:

- Product Design
- Industrial Design
- Interaction Design
- DFM
- Project Management
- UX Research
- UX Design
- UI Design
- Web Design
- Media Production
- Visual Storytelling
- Award / Competition
- Client / Industry Project
- Team Project
- Individual Project

## Flexible Case Study Requirement

Do not force every project into the same fixed case-study structure.

Different projects need different sections.

Product design projects may focus on concept, interaction, product story, design decisions, and final outcome.

Real client or DFM projects may focus on brief, challenge, constraints, project management, production considerations, and outcome.

UX projects may focus on research, insight, journey, prototype, testing, and final design.

Media projects may focus on brief, crew role, production process, story, and final output.

The website should support flexible content blocks such as:

- text section
- left image / right content block
- image gallery
- highlight card
- metric card
- role card
- process steps
- badge row
- company logo row
- quote highlight
- before / after
- video embed
- prototype link
- external link card
- PDF summary block

## Metrics Rule

Do not invent numbers.

Only show metric cards if real metrics are provided.

If no metrics exist, use qualitative outcome language.

## Layout Direction

Preferred project page layout:

- large hero image
- title, tags, role, year, project type
- optional award badge / company logo
- editorial sections
- left image / right content blocks where suitable
- alternating image-content sections
- short paragraphs
- highlighted key points
- strong image captions
- responsive mobile stacking

2. Responsive layout
3. Project cards
4. Optional award badge support

7. PDF export pages with print CSS
8. Placeholder project support
9. Easy future project updates
10. No database required

## Recommended Tech Stack

Use Astro with Markdown or MDX project files.

Use static site generation.

Use local images from public/images/projects/.

Use CSS for styling.

Do not add unnecessary backend, database, CMS, or complex animation framework.

## PDF Requirement

The PDF pages should be browser-print friendly.

Target PDF length:

- around 10 pages
- email-friendly size
- suitable for job applications

PDF variants:

1. Product / Industrial Design
2. UX / UI
3. Project Management / Design Coordination

## Future Maintenance Rule

Adding or updating a project should require:

1. adding images
2. editing one project content file
3. updating tags and PDF visibility
4. no major code change

## Important Do Not Invent Rule

Do not invent:

- clients
- awards
- research findings
- impact metrics
- business outcomes
- user quotes
- project details

Use placeholders where information is missing.
